# 포트폴리오 프로젝트 전체 정리

PC에 있는 프로젝트 폴더를 기준으로, 어떤 시스템을 개발했는지 정리한 문서입니다.

---

## 1. 어메이징사업부 (amazing-biz-server)

**폴더**: `c:\Users\ok\amazing-biz-server`

**한 줄**: 어메이징사업부 실적·리드·본부 관리 백오피스 + 리드 분배(DB 분배) 시스템

**기술 스택**: Node.js, Express, PostgreSQL, React(프론트), JWT 인증, 카카오 알림톡

**주요 기능**  
- **인증/권한**: 로그인·회원가입, 역할(어메이징/본부장/팀장 등) 기반 권한  
- **본부·팀 관리**: 본부/팀 CRUD, 공개 API  
- **실적(performance)**: 실적 등록·조회·집계  
- **리드(고객) 관리**: 리드 CRUD, 엑셀 업로드, 지역/본부별 분배(DB 분배), 담당자 할당, 계약 PDF  
- **목표(goals)**: 목표 설정·대시보드·현황  
- **캘린더**: 일정/이벤트 관리  
- **할일(todo)**  
- **공지(notices)**  
- **집계(aggregates)**: 성과 집계·백업  
- **알림**: 카카오 알림톡, 알림 스케줄러  
- **DB**: 스키마 자동 동기화, 마이그레이션 스크립트 다수  

**배포**: Railway 등 (GitHub Actions 자동 배포)

## 2. 블로그 어메이징사업부 (amazing-biz-blog)

**폴더**: `c:\Users\ok\amazing-biz-blog`

**한 줄**: 보험 영업 지원 플랫폼 — 블로그/카페용 Q&A 생성, 보험료 비교, 판례 기반 답변

**기술 스택**: Next.js 14+, TypeScript, Tailwind, Supabase(인증·DB), OpenAI/Gemini, Google APIs, pdf-parse, xlsx

**주요 기능**  
- **Q&A 생성기**: 상품명/질문 입력 → AI(GPT·Gemini)로 카페용 질문·답변 생성, 스레드형 대화  
- **보험 상품·보험료 데이터**: CSV/엑셀 기반 보험상품 마스터, 보험료 비교 시트, 질병분류표(KCD)  
- **판례/분쟁조정 사례**: PDF 판례집 파싱·업로드, 답변 시 참조  
- **계층적 권한**: 본부/팀/멤버십, RLS 정책  
- **할당량/사용량 관리**: API 할당량·초과 진단  
- **스크립트**: 보험료 데이터 업로드, 판례 파싱, 시트 생성  

**배포**: Vercel (blog.어메이징사업부.com 등)

## 3. 치매검사 든든이 (amazing-biz-brain)

**폴더**: `c:\Users\ok\amazing-biz-brain`

**한 줄**: 웹 기반 인지(치매) 선별 검사 — 15문항, 영역별 가중치 채점·결과 이메일 발송

**기술 스택**: Next.js 16, TypeScript, Tailwind, Nodemailer(이메일 발송)

**주요 기능**  
- **인지 검사 15문항**:  
  기억력(그림 기억·재인), 지남력(시계 맞추기), 계산력(거스름돈·연속 감산), 시공간(카드 짝맞추기), 집행기능(100-7), 억제능력(Stroop), 주의력(기호 개수·두더지 잡기), 작업기억(숫자 거꾸로), 반응속도, 시각탐색(Schulte), 시각추론(패턴), 언어유창성(단어 선택), 판단력(돌봄 질문)  
- **영역별 가중치 배점**: 기억력 25, 지남력 15 등 총 100점  
- **모바일 최적화**: 한 손 조작, 타이머·가이드 오버레이  
- **결과 이메일 발송**: API(`/api/send-email`)로 검사 결과 메일 전송  

**배포**: Vercel 등 (포트 3100 로컬)

- **멤버 라운지**: 배지 신청·업로드, 명함 신청·발송, 교육 신청  
- **관리**: 멤버 승인, 사용자 상태 관리, 이메일 발송  
- **API**: auth, members, send-recruit, send-business-card, upload-badge 등  

**배포**: 루트 도메인(어메이징사업부.com) 또는 서브도메인

## 5. 청구닷컴 (cheongu-landing)

**폴더**: `c:\Users\ok\cheongu-landing`

**한 줄**: 보험금 청구 링크 허브 — 손해/생명/공제 회사별 전산·청구서·고객센터·FAX 링크 + PDF 자동 입력

- **회사별 링크**: 삼성화재·현대해상·DB·KB·메리츠·롯데·흥국·하나·농협·카카오페이·캐롯·AIG·라이나·한화·교보·미래에셋·신한·NH·우체국·The·K·MG 등 손해/생명/공제 다수  
- **항목**: 전산 접속, 필요서류 안내, 보험금 청구서 PDF, 치과치료확인서 PDF, 고객센터·인콜 모니터링·헬프데스크·FAX 번호  
- **공통 PDF**: 입원/통원 청구서, 암 진단 확인서, 수술 확인서, 개인정보 동의서  
- **PDF 자동 입력**: AcroForm 템플릿 생성·검증 스크립트, 보험금 청구서 자동 채우기  
- **PWA**: manifest, 아이콘  

**배포**: 정적 호스팅 (청구.com 등)
